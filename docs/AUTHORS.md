### Contributing to VBDP

All contributors who make meaningful contributions to the project will be listed in this file. Contributions include:

- **Code**: Implementing features, fixing bugs

- **Design**: Architecture, UX/UI design

### Automatic Attribution

Contributors are automatically recognized through:

1. **Git Commits**: Your name from Git config appears in commit history
2. **Pull Requests**: GitHub tracks PR authors
3. **Issue Reports**: Bug reporters are credited in release notes
4. **Security Researchers**: Listed in Security Hall of Fame

If you've contributed but prefer not to be listed publicly, contact the maintainers.

## Acknowledgments

### Inspiration and Prior Work

VBDP builds on ideas and lessons learned from:

- **bsdiff/bspatch** by Colin Percival - Binary diffing algorithm
- **Chromium Courgette** - Executable-aware diffing
- **Google's update systems** - Large-scale update distribution
- **APT/DNF** - Package management and delta updates
- **Microsoft's differential download** - Binary delta compression
- **BDP (Binary Differential Patching)** by Mikhail Gusarov - Original inspiration and problem analysis

### Technology Stack

VBDP is built with:

- **Rust** - Primary implementation language
- **PostgreSQL** - Server database
- **SQLite** - Client/publisher storage
- **libsodium** - Cryptography (Ed25519)
- **Blake3** - Fast hashing
- **bsdiff** - Binary diffing

Thank you to all maintainers and contributors of these projects.
